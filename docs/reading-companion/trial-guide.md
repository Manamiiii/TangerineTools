# 阅读伴侣试用说明

## 可试用范围

阅读伴侣的无模型核心流程可以用于真实阅读测试：

- 从书架选择《飘》的指定版本。
- 设置实际读到的章节，进度保存在当前浏览器。
- 粘贴刚读到的一小段文字；段落只存在于当前页面，不上传、不持久化。
- 在本机扫描段落中实际出现的已审计名称；未命中的资料包候选不会显示。
- 由读者确认名称出现于当前章节，随后查看可用的地点分类和安全空间资料。
- 返回较早章节时，系统隐藏记录在较后章节的名称及其解锁资料。

截图 OCR、自由问答、语义实体识别、人物关系理解和未知名称发现不属于本次试用范围。

## 本地启动

环境需要 Node.js `>=20.19.0`。在仓库目录运行：

```bash
npm install
npm run dev
```

打开终端显示的本地地址，进入「经典文学阅读」场景。开发分支可以直接用于本地测试；公开 GitHub Pages 站点只部署 `main` 分支。

## 建议试用流程

1. 进入「经典文学阅读」，确认首先显示书架。
2. 选择《飘》，把进度设置为真实阅读章节。
3. 从正在阅读的位置复制一小段文字并执行“本机扫描已知名称”。
4. 核对扫描结果是否确实出现在原段落中；未命中时界面不应给出候选提示。
5. 对正确结果执行确认，检查名称是否记录在当前章节，以及地点资料是否按预期解锁。
6. 返回书架并重新进入，确认阅读进度与已确认名称仍然存在。
7. 暂时切换到较早章节，确认较后章节记录被隐藏；再恢复真实进度。

手动确认错误时，可以在“记录本章遇到的名字”区域删除对应记录。

## 试用反馈重点

- 从微信读书复制文字到工具的操作是否顺手。
- 章节选择是否足以表达当前进度。
- 精确扫描的漏识别频率是否影响使用。
- 地点分类、地图概览和范围说明是否真正帮助理解。
- 哪类问题最希望下一步由 OCR 或模型能力解决。

试用期间不要为了提高命中率输入尚未读到的名称。资料包不会主动列出按需实体，也不会根据前缀或近似拼写猜测候选。
