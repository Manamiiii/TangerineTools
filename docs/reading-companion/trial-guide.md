# 阅读伴侣试用说明

## Windows 首版可试用范围

阅读伴侣可以用于一次真实阅读测试：

- 从书架选择内置《飘》，或使用书名、作者、版本信息和章节目录创建个人书籍。
- 设置实际读到的章节；进度、已遇到名称和个人地图位置保存在当前浏览器。
- 使用“从剪贴板粘贴”、`Ctrl+V` 或截图输入当前小段。
- 截图使用设备端 Tesseract.js 执行简体中文和英文 OCR；结果只进入当前页面，必须先人工核对。
- 使用确定性本机扫描查找段落中实际出现的已审计名称。
- 可选配置 OpenAI Chat Completions 兼容接口、模型名和 API Key，让模型识别当前小段中的名称候选。候选必须逐项确认，不能自动生成正式事实。
- 将地点分类为不确定、现实、虚构、有现实原型或位置模糊。只有现实地点会开放公网地图搜索。
- 国际地图候选可携带 GeoJSON 点、河流/山脉路径或区域范围；国内天地图需要浏览器端 Key。
- 书内功能按“阅读输入 / 本章记录 / 地图 / 已读资料 / 设置”切换；地图支持大画布和全屏展示。
- 个人书籍可以从书架删除；确认删除后会一并清理该书所有场景的阅读状态。

自由剧情问答、自动人物关系、未来章节解释、移动端原生分享和 Windows 全局快捷键不属于当前能力。

## 启动与安装

环境需要 Node.js `>=20.19.0`。在仓库目录运行：

```bash
npm install
npm run dev
```

Windows PowerShell 如果阻止执行 `npm.ps1`，使用：

```powershell
npm.cmd install
npm.cmd run dev
```

打开终端显示的地址，进入「经典文学阅读」场景。正式 HTTPS 站点可在 Edge 或 Chrome 地址栏或“应用”菜单中安装为 TangerineTools 独立窗口；书架也会在浏览器提供安装事件时显示“安装应用”按钮。

## 建议试用流程

1. 进入「经典文学阅读」，确认先显示书架。
2. 选择《飘》，或创建一本只含三章的个人测试书。
3. 设置真实阅读章节。
4. 从微信读书复制一小段，点击“从剪贴板粘贴”；也可选择截图并执行“提取截图文字”。
5. 核对段落文字，再执行“本机扫描已知名称”。
6. 如需测试模型，进入“设置”，按内置教程填写接口、模型名和 API Key；确认界面清楚说明即将外发的范围，再回到“阅读输入”运行识别。
7. 逐项核对模型候选后确认。模型候选不应附带人物关系、剧情或未来事实。
8. 手动记录一个地点。保持“不确定”时不应出现公网搜索；改成“现实地点”后才出现搜索入口。
9. 进入“地图”，对真实河流、山脉或行政区选择候选，检查服务提供几何时是否显示路径或范围；没有几何时显示代表点，并验证“全屏地图”能铺满应用窗口。
10. 返回书架并重新进入，确认进度、名称分类和个人地图位置仍存在。
11. 暂时退回较早章节，确认较后章节记录及其地图位置被隐藏。

## 隐私与网络边界

- 段落、截图和 OCR 结果不持久化。
- 本机扫描与 OCR 不把内容发送到模型服务。
- 点击模型识别时，当前段落、书名和章节标签会发送到用户配置的接口。
- 模型 API Key 只保存在当前 `sessionStorage`；关闭浏览器后失效，不进入 IndexedDB、导出文件或仓库。
- 公网地图只接收用户主动搜索的现实地点名称；虚构、原型、模糊和不确定地点不发送。
- OCR 语言数据随应用发布；Tesseract.js 运行核心首次初始化可能仍需要网络，之后由浏览器缓存复用。

## 反馈重点

- 微信读书复制到工具的操作是否顺手。
- 中文截图 OCR 的速度和误识别是否可接受。
- 模型候选是否严格限制在当前段落，不添加剧透性解释。
- 地点分类能否阻止虚构地点误搜。
- 河流、山脉和区域的路径/范围是否比单点更有帮助。
- PWA 安装、开始菜单启动和本机数据保留是否符合 Windows 使用习惯。
