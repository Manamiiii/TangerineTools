# 经典文学阅读伴侣：产品与技术规格

## 状态与定位

本文定义 TangerineTools 中“经典文学阅读伴侣”的现有能力、内容建库方式、运行时边界、剧透安全规则和跨端技术边界。场景工具 `reader` 提供书架入口、个人书籍创建与删除、版本化资料读取、章节进度、Windows 剪贴板、设备端截图 OCR、本机已知名称扫描、可配置的窄范围模型候选识别、用户确认名称、按章节过滤、可全屏的真实地点互动地图、公网地图候选搜索、外部服务设置教程和确定性剧透门禁基础；应用预设「经典文学阅读」场景。正式《飘》章节实体与事实、历史地图图层、自由剧情问答、自动人物关系、Windows 全局快捷键和移动端原生分享入口仍属于待建设范围。

阅读伴侣面向“一次专注阅读一本书”的个人使用方式。使用者可以提前确定下一本书，因此系统以**按书预分析并构建版本化资料库**为主，运行时识别和大模型交互只承担辅助、消歧和补充解释，不追求在没有准备资料的情况下完整理解所有书籍。

核心目标是：

- 帮助读者理解地点、人物、时代背景和已读内容之间的关系。
- 地图类纯空间信息可以直接、宽松地展示。
- 任何存在合理剧透概率的回答或展示都必须在内容出现前给出强提醒。
- 读者明确授权后才生成或展示潜在剧透内容。
- 工具不注入、不逆向微信读书，也不依赖其私有接口。

## 使用范围

### 界面层级

进入「经典文学阅读」场景后先显示书架，不自动打开目录中的第一本书。选择书籍后进入该版本独立的阅读空间，书内只选择阅读章节；通过“返回书架”切换书籍。这样可以在增加新书时保持每本书的版本信息、进度和读者确认记录相互隔离。

书架中的“添加书籍”可以只凭书名、作者、版本信息和章节数或逐行目录建立个人空资料包。个人资料包保存在 Dexie `meta` 表，使用独立命名空间，不修改 schema；它只提供章节进度、读者记录和地图等通用系统能力，不会凭书目信息自动产生人物、事实或剧情边界。正式可分发资料仍必须经过 staging → preview → 显式发布流程。

书架同时展示首次试用路径和当前能力边界。Windows 首版支持安装为 PWA 独立窗口、剪贴板输入、截图 OCR 和可选模型候选识别；自由剧情问答、自动人物关系和移动端原生分享保持关闭。具体试用步骤见 [`trial-guide.md`](trial-guide.md)。

选书后的主空间使用“阅读输入 / 本章记录 / 地图 / 已读资料 / 设置”五个任务 Tab。章节边界与段落输入同属“阅读输入”；用户确认名称归入“本章记录”；地图使用独立整栏画布；模型和地图服务的配置、清除与教程统一位于“设置”。Tab 只改变当前任务视图，不改变章节门禁、资料状态或本机持久化语义。

### 目标终端

- Windows 是当前实现目标：使用 Edge 或 Chrome 访问后可安装为 PWA 独立窗口，从微信读书复制文字或选择截图。
- iPhone 和华为平板属于后续适配目标；当前网页仍可打开，但不承诺原生分享、安装和 OCR 体验。

### 内容入口

微信读书没有可依赖的第三方阅读器插件接口，因此阅读伴侣作为独立工具接收用户主动提供的内容：

1. 优先接收微信读书选中文字后的复制或系统分享。
2. 无法分享文字时，接收当前页或当前段落截图并在设备端 OCR。
3. Windows 支持浏览器剪贴板按钮、`Ctrl+V` 和截图文件选择；全局快捷键及直接区域截屏仍需要原生壳。
4. iOS 通过 Share Extension 接收文字、图片或链接。
5. 华为 Android 设备通过 Android Share Target 接收内容；纯 HarmonyOS 通过 ArkTS 分享接收能力适配。

系统不能拦截微信读书原生页面里的地名点击。接近原始设想的交互是：用户分享或截取当前页后，工具在识别结果上框出地名，用户点击识别出的地名查看地图。

## 按书提前建库

### 当前书籍版本

运行时目录包含《飘》的指定中文版本：

- 作品：玛格丽特·米切尔《飘》（Gone with the Wind）。
- 译者：范纯海、夏旻。
- 出版社：长江文艺出版社。
- 出版时间：2018 年 5 月。
- ISBN：`9787570202188`。
- 数字阅读入口：微信读书。
- 章节边界：连续 1–63 章，稳定 id 为 `chapter-01` 至 `chapter-63`。

版本信息来源于使用者提供的微信读书详情和版权信息。纸质上下册分界不作为数字阅读进度边界。资料包的实体、事实、来源和地图字段必须经过后续审计才可加入正式集合。

### 建库输入

准备下一本书时创建一个独立书籍资料包，至少记录：

- 书名、作者、原作语言和作品标识。
- 阅读版本、译者、出版社或电子版特征。
- 章节目录及章节稳定标识。
- 故事时间范围和主要地理范围。
- 可合法使用的参考文本、公开资料和地图资料来源。

不同译本可能使用不同的人名、地名和章节划分，资料包必须区分版本；无法确定版本时应降低章节定位置信度并采用更保守的剧透策略。

### 预分析产物

大模型可以在建库阶段生成候选内容，但候选必须结构化、可追溯并允许人工修正。资料包包含：

- 地点实体、别名、原文名、译名和地理层级。
- 真实地点坐标、历史名称和适用时间范围。
- 虚构地点、原型地点或模糊区域及其置信度，禁止伪造精确坐标。
- 地点首次出现章节和各章节已经揭示的地点事实。
- 人物、别名、称呼及按阅读进度逐步揭示的关系。
- 历史事件、时代概念及其与作品剧情的隔离标记。
- 已审计的地图图层、路线和时间线片段。
- 每条事实的来源、风险级别、最早可揭示进度和风险类别。

建议的数据单元包含：

```json
{
  "id": "place-stable-id",
  "name": "地点名称",
  "kind": "place",
  "placeKind": "real | fictional | prototype | approximate",
  "aliases": [],
  "parentLabel": "地理层级说明",
  "revealAt": {
    "chapterId": "chapter-12"
  },
  "sourceIds": ["source-stable-id"],
  "geometry": {
    "type": "point | area",
    "latitude": 0,
    "longitude": 0,
    "radiusKm": 20
  }
}
```

每个正式实体必须引用已批准来源和指定版本中的首次可揭示章节。真实地点或有来源支持的原型地点可以使用 `point`；模糊区域使用带半径的 `area`；虚构地点不得通过 `point` 伪造精确坐标。无章节证据或无批准来源的候选不进入正式资料包。

事实数据单元包含：

```json
{
  "id": "fact-stable-id",
  "bookId": "gone-with-the-wind",
  "editionId": "edition-stable-id",
  "entityIds": ["place-atlanta"],
  "kind": "spatial | character | plot | history | concept",
  "content": "结构化事实或展示数据",
  "revealAt": {
    "chapterId": "chapter-12",
    "anchor": null
  },
  "riskLevel": "safe | potential | high",
  "riskCategories": ["location_significance"],
  "sourceIds": ["source-stable-id"],
  "confidence": 0.96
}
```

每条正式事实必须包含类型、内容、实体引用数组、已批准来源、指定版本中的可揭示章节、风险级别和风险类别。风险类别只能使用系统预定义枚举，警告界面不读取模型生成的自由文本。`revealAt` 是可展示边界，不代表内容一定应自动展示；`potential` 和 `high` 仍需经过剧透门禁。

### 建库流程

```text
输入书目与版本
  → 收集合法参考资料
  → 划分章节与稳定阅读进度
  → 大模型批量提取实体和候选事实
  → 地名数据库校验坐标和别名
  → 标注 revealAt、风险等级和来源
  → 人工抽查高风险事实与虚构地点
  → 生成版本化资料包
  → 预览报告确认后发布
```

资料包应采用类似现有 BWiki 预置的 staging → preview → 显式发布思路。运行时只读取已发布资料，不直接把未经检查的大模型批量结果当作正式内容。

当前发布目录与命令为：

- `scripts/reading-companion/data/staging/`：版本化研究输入、来源状态、实体和事实候选。
- `scripts/reading-companion/data/preview/`：由 staging 生成的逐书完整发布候选、统一 catalog 和审计元数据。
- `npm run preview:reader`：生成 preview。
- `npm run check:reader:preset`：比较 preview 与正式资料并在 `artifacts/reading-companion/` 生成报告。
- `READING_PACKAGE_OVERWRITE=CONFIRM_READING_PACKAGE npm run apply:reader:preset`：显式写入正式资料包。
- `npm run check:reader:packages`：校验运行时目录和正式资料包。

资料源状态使用 `approved`、`candidate` 和 `rejected`。只有 `approved` 来源可以进入运行时资料包；候选和拒绝项只保留在 staging 与 preview 审计元数据中。

实体与事实研究结果分别保存在 `entityCandidates` 和 `factCandidates`。每项都包含候选状态、结构化数据、引用来源、阻塞项和审阅备注。`candidate` 必须至少声明一个阻塞项；候选数组不会被发布管线隐式合并到正式 `entities` 或 `facts`，只有人工核对来源、指定译本别名和 `revealAt` 后，才可显式移入正式集合。preview 和检查报告会列出候选 id 与阻塞项，但运行时资料包不包含候选内容。

管线枚举 staging 目录内的全部 JSON，不保存单一书籍硬编码。每份 staging 都声明唯一 `slug`、顺序、catalog 展示信息、版本、来源状态、实体和事实：

- 首次发布的新书必须提供完整 `package`，包括作品、版本和稳定章节。
- 已有书更新可以提供 `basePackagePath`，在正式包的作品、版本和章节基础上替换受审计的来源、实体和事实。
- preview 自动生成 `presets/reading-companion/{slug}.json` 目标和统一 catalog 条目。
- slug、顺序、资料包 id 和 catalog 路径必须唯一且满足固定路径格式。

因此，新书接入的数据准备不要求修改浏览器组件或发布脚本；建库质量仍通过首本真实书籍的完整流程逐步校准。

### 当前资料源候选

- 使用者提供的微信读书版本和版权信息是已批准来源，只用于版本、译者、ISBN 和 1–63 章结构。
- [Library of Congress · America Reads](https://www.loc.gov/exhibits/america-reads/1900-to-1949.html#obj019) 用于候选原作书目和作品历史范围，不用于中文译文章节定位。
- [U.S. Census Bureau · Gazetteer Files](https://www.census.gov/geographies/reference-files/time-series/geo/gazetteer-files.2025.html) 用于候选现代行政区、GEOID 和代表坐标；不能证明作品年代的边界或交通状况。
- [U.S. Census Bureau · Georgia Incorporated Places](https://tigerweb.geo.census.gov/tigerwebmain/Files/acs25/tigerweb_acs25_incplace_ga.html) 用于核对亚特兰大和琼斯伯勒的现代代表坐标。
- [Atlanta History Center](https://www.atlantahistorycenter.com/blog/so-red-the-rose-the-gone-with-the-wind-that-never-was/) 用于区分真实的亚特兰大地点与虚构种植园，并防止给塔拉等虚构地点伪造精确坐标。
- [New Georgia Encyclopedia · Gone With the Wind](https://www.georgiaencyclopedia.org/articles/arts-culture/gone-with-the-wind-novel/) 与 [Clayton County](https://www.georgiaencyclopedia.org/articles/counties-cities-neighborhoods/clayton-county/) 用于候选时代范围、地点名称和地区背景。包含剧情概述的页面仅供建库审阅，不能直接成为运行时安全内容。
- [Library of Congress · General Maps](https://www.loc.gov/collections/general-maps/about-this-collection/rights-and-access/) 用于候选历史地图。每个地图条目必须单独检查 Rights Advisory 与署名要求。
- 互动地图支持两种可切换底图：国际模式使用 [OpenStreetMap](https://www.openstreetmap.org/copyright) 在线标准瓦片，免 Key，但在国内网络可能需要 VPN；国内模式使用[天地图](https://www.tianditu.gov.cn/)矢量底图与注记，需要使用者在设置中填写浏览器端 Key。选择和 Key 只保存在当前浏览器的 `localStorage`，不进入资料包、IndexedDB 导出或仓库。设置页提供天地图控制台入口、浏览器端应用申请步骤、清除 Key 和回到地图验证的说明。两种模式都始终显示供应商署名，不提供瓦片预取或离线下载。底图配置集中在 `src/features/reading-companion/map/mapConfig.js`，底图只负责显示，不能成为正式地点坐标或虚构地点定位的证据。
- 资料包外地点可以由读者主动发起公网地图搜索。国际模式使用 Nominatim，限制为点击后搜索、最多五条结果、每秒最多一次实际请求并缓存会话内重复查询；国内模式使用天地图地名搜索并复用浏览器端 Key。界面会在请求前说明搜索词将发送给所选服务，不提供自动完成或后台批量查询。
- 公网搜索结果只是现代地图候选。新记录的地点默认为“不确定”，只有读者明确分类为现实地点后才允许发起公网搜索；虚构、原型和模糊地点不会发送给地图服务。读者确认某条结果后，系统把点坐标或服务返回的 GeoJSON 点、线、面几何作为 `observedEntities[].mapLocation` 保存到个人阅读状态。河流、山脉和行政区在地图服务提供几何时按路径或范围展示；缺少几何时只显示代表点。该位置随章节边界显示，标记为个人确认内容，不写回正式资料包，也不能用于给虚构或含糊地点强行定位。

这些外部来源均不能替代指定中文译本对 `revealAt` 的章节证据。缺少对应章节文本时，实体和事实保持待审，不进入正式资料包。

没有可用原文时，章节仍有两项确定用途：

- `currentChapterId` 表示读者实际进度，负责隐藏较后章节的本地记录并为未来检索限定上界。
- `observedEntities[].firstSeenChapterId` 表示读者主动确认“最晚已在这一章遇到该名称”。它是个人阅读锚点，不宣称这是作品中的客观首次出现章节，也不会自动批准候选实体、坐标、人物关系或剧情事实。

正式资料包的 `revealAt` 与本地 `firstSeenChapterId` 不可互相替代。前者需要指定版本证据并可供所有使用者复用；后者只来自当前读者输入，保存在当前设备，可在没有全文、OCR 和模型时工作。读者退回较早章节时，系统隐藏锚定在较后章节的名称，避免本地记录本身造成剧透。

资料包可以另带 `onDemandEntities`，用于无原文条件下的按需精确匹配。该集合遵守以下安全边界：

- 不在界面中列出候选、提供自动完成或做模糊搜索。
- 只有用户已经输入同名且同类型的名称后才返回结果；匹配对全角/半角、首尾空格和英文大小写做规范化，但不允许前缀或近似匹配。
- 只包含已批准来源支持的名称、真实/虚构分类、地理层级、现代代表坐标和范围说明，不包含人物关系、地点剧情意义或事件结果。
- 解锁章节取自本地 `firstSeenChapterId`，界面明确标记为“读者确认后精确解锁”，不把它冒充指定译本的客观首次出现章节。
- 真实地点坐标可以进入互动地图；虚构地点不伪造精确坐标。退回较早章节时，按需实体和名称一起隐藏。

不得绕过微信读书鉴权、反自动化措施或私有接口批量获取作品全文。原文和参考资料的获取、保存与模型传输必须符合其授权范围；运行时只处理用户主动提供的最小必要段落。

### 能力归属

- **系统能力**：仓库中的确定性代码、资料包 schema、校验器、章节过滤、空间投影、剧透规则、发布命令和测试。它们可在没有当前开发会话和模型的情况下重复运行。
- **建库候选**：开发时由 Codex、其他模型或人工研究形成的实体、事实和来源建议。它们必须保存在 staging，并在来源与章节边界获批后才能成为系统正式数据。
- **运行时模型能力**：应用提供 OpenAI Chat Completions 兼容适配器，用户配置 HTTPS 接口、模型名和 API Key 后，只把当前小段、书名和章节标签发送给该服务。返回值只允许成为人物、地点、概念或事件名称候选，必须由读者逐项确认；不能写入正式资料包、生成事实或绕过章节门禁。API Key 只存在当前 `sessionStorage`，接口地址和模型名保存在 `localStorage`。

测试夹具只证明系统可以处理某种数据结构，不代表对应书籍、地点或事实已经进入正式资料库。

## 运行时能力

### 无模型也可工作的能力

- 从 Windows 剪贴板读取当前小段，或使用 Tesseract.js 在设备端对中英文截图执行 OCR；OCR 语言数据随应用发布，识别文字只进入当前页面并需人工核对。
- 根据正式资料包和当前章节确定性过滤可见实体。
- 在 Leaflet 互动地图中展示已审计地点，支持拖动、缩放、触控、列表与标记联动和多地点视野适配；OpenStreetMap 在线瓦片不可用时，地点标记、坐标和详情仍可降级展示。
- 计算已读地点之间的方向和球面直线距离；结果只使用已批准经纬度，不推断路线、交通方式或剧情移动。
- 标记真实地点、虚构地点、原型地点和模糊区域。
- 查询书籍资料包内的别名、译名和已审计说明。
- 按当前阅读进度过滤人物、地点、关系和时间线。
- 保存当前书籍、版本、章节和已查看实体。
- 记录读者在当前章节主动确认遇到的人物、地点、概念或事件名称；同名同类型记录去重，并保留更早的确认章节。
- 将读者已确认名称与资料包中的窄范围公开候选做精确匹配，并在匹配后显示经过来源审计的安全分类与空间字段。
- 扫描用户主动粘贴的当前段落，只返回文本中实际出现的 `onDemandEntities` 名称或别名；英文匹配要求完整词边界。扫描结果仍需用户确认才写入已读记忆，不会枚举未出现候选或推断未知实体。
- 对资料包未收录但读者已经记录的地点发起显式公网地图搜索，并在读者选择某个现实地点候选后保存个人地图位置；搜索服务不会判断作品里的地点身份，未经确认的结果不会进入地图。

### 运行时大模型的辅助职责

- 当前适配器只从当前段落或 OCR 结果中识别原样出现的人物、地点、概念和事件名称。
- 模型不能直接确认现实地点、生成关系、事实或剧情解释；地点无法由当前段落确定时必须保持“不确定”。
- 同名消歧、低置信度补充解释和自由问答属于后续能力，并继续受双阶段剧透门禁约束。

运行时大模型不是坐标、阅读进度或剧透授权的最终决定者。确定性代码负责：

- 当前阅读边界。
- 允许检索的资料范围。
- 地图坐标和距离。
- 风险级别触发条件。
- 是否显示确认界面。
- 用户是否授权查看潜在剧透。

### 已读记忆

即使书籍已经预建库，系统仍保存一份本地已读状态：

- 当前书籍和版本。
- 已确认章节及可选的段落锚点。
- 已经在阅读内容中出现的实体。
- 用户主动输入的名称及其首次确认章节；该记录不附带未经审计的外部事实。
- 用户主动确认或纠正过的地名映射。
- 用户对剧透内容的单次授权，不默认转化为全局授权。

资料包定义“经过版本证据审核后，作品资料何时可以揭示”；已读记忆定义“当前读者实际读到哪里，以及主动确认遇到了什么”。正式资料按两者交集过滤，本地名称按其用户确认章节过滤。

## 地图与理解辅助

### 地图

- Leaflet 负责浏览器内的拖动、缩放、触控、标记与视野控制。
- OpenStreetMap 或天地图瓦片提供在线现代地理背景并显示供应商署名；瓦片服务没有离线或可用性保证。
- 地图使用独立任务 Tab、整栏大画布和全屏按钮。优先请求浏览器原生全屏；原生能力不可用或被拒绝时使用覆盖当前窗口的全屏回退，`ResizeObserver` 保证 Leaflet 在尺寸切换后重新铺满容器。
- 真实地点使用已批准坐标显示点标记，模糊区域使用带半径的范围，地点列表与地图选择联动。
- 读者从公网地图搜索结果中确认的现实地点使用独立标签显示，只属于个人已读记忆，不等同于已批准坐标。
- 多个已读地点之间展示方向和球面直线距离，不把它解释为历史道路距离或人物行程。
- 虚构地点没有可靠几何信息时只显示地点卡片，不伪造现实地图坐标。

故事年代适用的名称、边界、交通方式、历史地图覆盖层和已读路线仍需独立来源及版本边界审核。

地图面板把纯空间信息和剧情解释分开。坐标、距离等安全字段可以立即出现；“为什么重要”“谁将在这里做什么”等内容进入剧透门禁。

### 人物与时代背景

- 当前进度内的人物别名、称呼和已经明确的关系。
- 人物上一次在已读范围内出现的地点。
- 时代制度、社会概念和历史背景。
- 作品事实与现实历史使用不同标签。

现实历史也可能暗示剧情。只要历史说明与作品人物、地点后续命运建立联系，就按潜在剧透处理。

## 剧透安全契约

### 基本原则

- 地图类纯空间事实使用明确白名单，可以直接展示。
- 只要存在合理概率涉及剧透，就必须先警告，后生成或展示。
- 无法确定风险时按潜在剧透处理。
- 警告本身不得包含具体剧情事实。
- 用户即使主动询问结局或后续剧情，仍需经过强提醒。
- 不使用“先生成、再模糊遮挡”的方式保护内容。

### 风险分级

#### `safe`

默认允许直接展示：

- 经纬度、方向、距离和行政层级。
- 不涉及后续命运的真实或虚构地点类型。
- 故事时间点之前已经成立的纯地理事实。
- 与剧情意义隔离的地图底图和历史边界。

安全白名单只适用于具体字段，不自动覆盖整个地图弹窗。

#### `potential`

展示前必须强提醒：

- 地点在作品中的重要性。
- 人物为什么前往或离开某地。
- 人物关系、动机和立场解释。
- 可能超过当前进度的历史与剧情联系。
- 路线、时间线或关系图中尚未确认安全的节点和边。
- 模型生成且不能证明完全来自已读范围的内容。
- 当前章节、版本或事实揭示边界不明确的内容。

#### `high`

展示前必须二次确认：

- 结局。
- 人物死亡、婚姻结果或最终命运。
- 身份反转和核心秘密。
- 明确发生在当前进度之后的关键剧情。

### 强提醒交互

潜在剧透警告只显示无细节的风险类别：

```text
⚠️ 以下内容可能涉及剧透

可能涉及：人物关系、地点的后续意义或当前进度之后的事件。
当前阅读进度：第 12 章。

[保持隐藏] [仍然查看]
```

- “保持隐藏”是没有运行时模型时的默认操作，且不会把事实内容写入页面。
- “仍然查看”使用明显警示样式。
- `high` 内容在“仍然查看”后增加第二次确认。
- 授权默认只对当前一次内容有效。
- 警告原因只使用预定义类别，不让模型写可能泄露剧情的自由文本。
- 接入运行时模型后可以增加“只生成严格无剧透版”，但生成结果仍需经过展示前扫描。

### 双阶段门禁

```text
用户请求或即将展示的组件
  → 检查当前版本和阅读进度
  → 只检索进度内资料
  → 前置风险分类
      ├─ safe：允许生成
      └─ potential/high：停止并先提示
  → 用户选择严格版或授权版
  → 生成候选内容
  → 展示前二次扫描
      ├─ 通过：展示
      └─ 风险升级：丢弃候选并重新提示
```

流式输出只能在前置门禁通过后开始。生成后的候选在二次扫描通过前不得写入可见 DOM、通知、无障碍文本或预览摘要。

### 覆盖范围

剧透门禁覆盖所有可能呈现内容的入口：

- 聊天回答。
- 地图标记、弹窗和路线。
- 人物关系图的节点与连线。
- 时间线。
- 搜索建议和“你可能还想问”。
- 图片标题、来源标题和摘要。
- 已读总结和通知。

## 与 TangerineTools 的集成

### 可复用部分

- React 19 + Vite 主界面。
- PWA 安装与静态部署。
- Dexie/IndexedDB 本地阅读状态。
- 场景和懒加载工具框架。
- 通用资料表、引用字段和 JSON 导入导出。
- 现有纯 CSS 和 hash 路由约定。

阅读伴侣适合作为新的专用工具，例如内部值 `reader`。经典文学场景可以同时启用通用资料库和阅读伴侣；地图、剧透门禁和问答使用专用组件，不把业务判断塞进通用资料表组件。

阅读伴侣运行时代码集中在 `src/features/reading-companion/`，维护脚本、版本化研究数据和专用测试集中在 `scripts/reading-companion/`。公共目录只承担工具注册、Dexie 实例、播种编排、兼容导出和全局样式；专用样式因项目约定保留在 `src/styles.css`，并使用 `reader-` 前缀隔离。

### 数据边界

当前 Dexie schema 保持 v1。MVP 优先使用现有场景、资料表、资料行和 `meta` 存储可表达的数据，不因规划文档直接引入 schema 版本变更。若专用阅读进度表、缓存表或授权记录确实需要新索引，必须另行设计并得到用户明确授权。

书籍正式资料包适合存放为版本化静态 JSON；读者进度、实体纠正和单次偏好保存在 IndexedDB。用户数据不得被预置更新覆盖。

### 大模型服务边界

现有 TangerineTools 是纯静态应用，模型供应商密钥不能写入仓库或公开构建产物。实现时在以下方式中明确选择：

1. 个人 BYOK：密钥只保存在用户设备本地，简单但跨设备和密钥保护能力有限。
2. 独立的轻量服务或 Serverless 代理：密钥留在服务端，适合正式跨端使用，但会引入后端、鉴权和隐私策略。
3. Windows 本地模型：隐私较好，但不能自然覆盖 iPhone 和华为平板。

模型访问通过独立适配接口提供，主界面和资料包不绑定具体供应商：

```text
analyzeExcerpt
resolveEntities
classifySpoilerRisk
generateExplanation
scanGeneratedContent
```

### 跨端外壳

```text
共享 React/PWA
├── 阅读伴侣主界面
├── 地图
├── 已读记忆
└── 剧透确认与安全展示

平台入口
├── Windows：Tauri 托盘、快捷键和区域截图
├── iOS：Swift Share Extension
├── 华为 Android：Android Share Target
└── 纯 HarmonyOS：ArkTS 分享入口
```

首个版本可以只实现 PWA 中的粘贴和截图上传，以验证按书建库、地名映射和剧透门禁；平台原生入口不应阻塞核心体验验证。

## 首本书验收范围

《飘》适合作为首个资料包。首版至少验收：

- 能区分亚特兰大等真实地点与塔拉庄园等虚构地点。
- 能展示主要地点的方向、距离和层级关系。
- 能根据指定译本识别常见人名、地名及别名。
- 当前进度之前的地点和人物事实可以正确过滤。
- 地图安全字段无需警告即可展示。
- 地点意义、人物关系和后续事件在展示前触发强提醒。
- 高风险问题必须经过二次确认。
- 生成后扫描能够拦截意外出现的未来剧情。
- 没有运行时大模型时，已发布地图和资料查询仍可工作。

## 建设顺序

1. 使用指定译本的最小章节证据审核《飘》staging 中的实体与事实候选，并显式发布首批章节资料。
2. 扩充无剧情地点覆盖，并审核模糊区域、原型地点和历史地图图层。
3. 用真实阅读流程校准截图 OCR、模型名称候选的误识别率和确认体验。
4. 在双阶段门禁与生成后扫描可验证前，不开放模型自由问答、关系解释或事实生成。
5. Windows PWA 流程稳定后，再评估全局快捷键原生壳、iOS 和华为原生分享入口。

## 待确认的集成选择

- 华为平板型号及 HarmonyOS 版本。
- 长期瓦片服务是否继续使用 OpenStreetMap 公共服务或切换为自有额度的供应商。
- 建库时允许使用的原文或参考资料来源。
- 当前 BYOK 浏览器直连是否长期保留，或在自由问答范围扩大前改用轻量后端或本地模型。
