# 性格推荐规则

本文描述性格推荐的当前行为和维护边界。运行逻辑以 `src/domain/nature.js` 为准，PVE 展示逻辑位于 `src/domain/naturePve.js`，资料适配位于 `src/domain/natureRowAdapter.js`。

## 目标

- 对全部 30 个合法修正性格给出推荐、可保留或不推荐结论。
- 先识别精灵定位，再评价强化收益、牺牲代价和同组支配关系。
- 每个结论提供可检查的理由、风险、六维变化和技能线索。
- 捕捉性格分档与 PVE 培养投入分别判断。
- 保持本地即时计算，不把推荐结果写回资料库或收集记录。

## 输入

必需输入：

- 生命、物攻、魔攻、物防、魔防、速度。
- 由一个强化维度和一个不同的弱化维度组成的性格；六个维度共形成 30 个候选。

洛克王国资料输入：

- `traitTags`：特性对应的输出、速度、耐久、回复、能量、控制等倾向。
- `skillRefs`：指向「技能资料」的引用。
- 技能的分类、威力、能耗、效果文本和 `effectTags`。
- 同编号全部有效形态的六维、系别、特性与技能，分别作为各自形态的培养和风险判断输入；一个形态的特性与技能不得混入另一个形态。

缺少特性或技能资料时，系统仍可基于六维给出基础判断，并明确降低解释置信度。

## 资料选择规则

- 精灵选择器每个编号只展示一个普通形态入口，隐藏非末端 X 阶成长形态、地区/外观变体和首领形态；标记为 X 阶但位于进化链末端、或同编号后续仅有首领形态的普通精灵可作为入口。同编号全部形态仍参与对比与综合分析。
- 精灵选择器使用一个组合框完成展开、按编号/名称/形态输入筛选和选择；再次展开时自动滚动到当前精灵。
- 普通入口对应的形态决定页面展示的基础六维。
- 选择精灵后只展示该精灵实际具有的特性标签；未选择精灵时不展示具体标签。
- 所选普通形态是展示入口；同编号成长、地区和首领形态各自独立分析。综合档位取各形态结果的并集：任一形态推荐则综合为推荐，否则任一形态可保留则综合为可保留；详情分别记录该性格适用的形态。
- 默认收起的形态对比逐项展示系别、六维、特性名称/图标/说明和完整技能组，并在摘要提示核心资料是否存在明显差异。六维变化以选择器中的普通入口形态为基准；普通形态与对应首领形态按一行两列成对展示，包括没有外观括号的基础形态（如岚鸟与霜翼领主）。
- 技能正文只从「技能资料」读取，精灵行只保存 `skillRefs`，不复制技能长文本。

## 判断层级

### 1. 六维分布

系统计算总和、均值、相对强弱、最高项、最低项、攻击差和耐久结构。所有六维条形图从 0 起算，以资料库全部有效形态、全部六维中的真实最高值作为统一满刻度，并以全局最低有效值作为固定参考线；相同数值在不同维度、精灵和页面中始终等长，切换精灵时刻度保持不变。顶部同时展示分位、排名和四分位统计；这些相对统计不参与条形长度计算。

### 2. 技能与特性证据

系统汇总物攻、魔攻和状态技能数量，结合攻击技能平均威力、路线占比与效果标签形成轻量证据。效果标签覆盖先手、速度、回复、减伤、能量、强化、削弱、控制、应对、轮转、连击、蓄力和场地等机制。

技能数量和公式辅助指标只用于定位校准，不视为实战伤害模拟。互斥技能、血脉选择和具体配招在资料缺少稳定分组字段时只作为风险提示。

物攻/魔攻面板差不超过 12、两侧面板达到实际输出基础且各有至少 4 个攻击技能时，弱化任一攻击侧本身不能成为不推荐理由。技能数量、基础威力和公式辅助差异只调整分数、优先级与风险说明，不再把另一侧硬支配为不推荐。该双攻保底不覆盖明确依赖某一攻击项的特性机制，也不覆盖低关注速度：基础速度未进入竞争圈、定位也不依赖抢速时，仍不为加速牺牲攻击项。功能、辅助或站场形态的最高物防/魔防达到全体上四分位，或者距离上四分位不超过 5 且比另一防御高至少 20 时，该突出耐久项属于路线基础，不作为常规牺牲项。

### 3. 定位识别

定位由六维、特性和技能共同决定，主要包括：

- 物攻输出、魔攻输出、双攻输出。
- 高速先手、条件速度、后手路线。
- 耐久站场、物理防御手、魔法防御手。
- 回复辅助、异常控制、能量循环、换入返场和强化成长。

单一标签不能独立决定定位；相互冲突的证据会降低推荐强度并进入风险说明。

### 4. 候选评价

强化项服务核心定位、提高主攻、关键速度或有效耐久时获得正向理由。弱化明确弃用的攻击项通常属于低代价牺牲。

以下情况会降低分档或形成风险：

- 弱化主攻、关键速度、生命或站场所需防御。
- 低速非速度定位为了加速而牺牲核心项。
- 站场定位继续降低当前耐久短板。
- 双攻证据不足时牺牲仍有实际用途的另一攻击项。
- 只由少量通用技能触发的机制被误当成核心定位。

同一强化项内存在明显更低代价的牺牲项时，较差候选受支配并进入不推荐。弃用攻击项形成的低代价牺牲优先于普通专项防御或速度豁免；“专项防御”不能单独豁免一个明显更差的攻击牺牲项。当同组已有明显更优的降非主攻方案时，不保留仅因高面板跨过阈值而出现的“强化生命、弱化一侧防御”候选。

功能型双攻、低输出功能位和护盾/免死触发路线可以保留有明确玩法依据的分支，但不会仅凭面板接近或标签数量进入推荐。双攻面板接近且两侧均有实际技能路线时，两侧攻击牺牲默认都可保留；技能池明显偏向一侧时，优势侧仍获得更高分，但不能仅凭当前不完整的技能数量、耗能和条件威力分析淘汰另一侧。全技能平均威力容易受蓄力、自损和强条件技能影响，不能单独把另一侧判为明显低价值。明确造成物理/魔法伤害或按对应攻击结算的特性仍保护其依赖攻击项。

## 分档

- 推荐：强化核心定位，牺牲项代价低，没有明显硬风险，也没有被同强化项更优候选支配。
- 可保留：具有明确玩法依据，但存在场景依赖、资料不完整或可接受风险。
- 不推荐：损害核心能力、放大关键短板、缺乏定位依据，或被同强化项明显更优候选支配。

`可保留` 面向实际捕捉取舍，允许保守保留有合理玩法的分支，但不收纳没有证据的劣解。

## 速度与耐久

- 速度收益取决于基础速度、速度定位、先手/迅捷机制和牺牲代价。
- 低速功能位的加速只在低代价且有明确节奏收益时保留。
- 高速输出位可以同时存在主攻强化与速度强化两种有效方向。
- 生命同时影响两侧承伤；单防强化需要结合生命、另一防御和站场机制判断。
- 专项防御可以形成保留分支，但不能掩盖弱化明显弃用攻击项的更优选择。

## PVE 培养投入

PVE 评级先判断精灵本体机制，再从推荐或可保留性格中选择培养方向。评级证据包括：

- 高面板且路线集中的输出能力。
- DOT、百分比伤害、真实伤害或稳定层数机制。
- 能量循环、队伍插件、强化传递、站场续航和控制价值。
- 攻击技能占比、平均威力和机制在 Boss 场景中的适用性。

推荐性格数量不直接决定培养价值。高速主攻位可以同时展示速度和主攻方向；普通功能或防御站场默认按需判断，只有明确高回报机制才进入更高投入档。

DOT 证据必须命中明确的持续伤害机制并达到足够覆盖度；泛化层数文本、少量通用中毒或火系技能不能单独构成 DOT 定位。

## 输出与解释

- 全部候选先按强化维度分组，再按推荐、可保留、不推荐分档。
- 每个候选按“同进化链精灵引用 + 候选性格值”匹配收集记录，在评分前显示该具体性格的已获得或未获得状态；非最终形态记录同样计入，未获得候选可打开预填精灵与性格的新增收集记录表单。
- 候选详情展示修正前后六维、个性理由、风险、定位和技能摘要。多形态精灵保留每个形态的独立评级与分数；候选行展示评分范围，详情逐形态展示评级和分数，不计算跨形态平均分。
- PVE 区按同编号全部有效形态分别展示五档星级、定位、主属性、主性格和必要的机制标签。
- 固定分析依据位于 PVE 与性格候选之间；性格切换时共用同一份依据，形态间技能或速度不同时另外列出各形态依据。
- 形态技能对比优先标出仅该形态拥有的技能；完整技能、速度线和规则原因默认可折叠查看。
- 不推荐且未获得的候选不展示获得状态或新增入口；推荐与可保留候选将“未获得”状态直接作为快速新增入口，所有候选保持相同宽度，适用形态展示在所选性格的右侧解析详情中。

## 维护与验证

规则调整必须：

1. 更新本文件中的当前规则描述。
2. 运行 `npm run check:nature`，检查 `artifacts/nature/calibration-report.md`。
3. 对 `confirmed-results.md` 中的确认结果做回归检查。
4. 按影响范围运行 `npm test`、`npm run lint` 和 `npm run build`。
5. 将尚未形成稳定规则的问题记录到 `open-issues.md`；解决后从问题文件移除。

不为单只精灵添加硬编码特判。外部资料只用于定位和机制核对，不直接提供最终推荐性格，也不使用旧网页游戏资料作为洛克王国世界的定位证据。
