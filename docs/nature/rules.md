# 性格推荐规则

本文描述性格推荐的当前行为和维护边界。运行逻辑以 `src/domain/nature.js` 为准，PVE 展示逻辑位于 `src/domain/naturePve.js`，资料适配位于 `src/domain/natureRowAdapter.js`。

## 目标

- 对全部 30 个合法修正性格给出推荐、可保留或不推荐结论。
- 先识别精灵定位，再评价强化收益、牺牲代价和同组支配关系。
- 每个结论提供可检查的理由、风险、六维变化和技能线索。
- 捕捉性格分档与 PVE 培养投入分别判断。
- 保持本地即时计算，不把推荐结果写回资料库或收集记录。

## 输入

必需输入：

- 生命、物攻、魔攻、物防、魔防、速度。
- 由一个强化维度和一个不同的弱化维度组成的性格；六个维度共形成 30 个候选。

洛克王国资料输入：

- `traitTags`：特性对应的输出、速度、耐久、回复、能量、控制等倾向。
- `skillRefs`：指向「技能资料」的引用。
- 技能的分类、威力、能耗、效果文本和 `effectTags`。
- 同编号关联首领形态的六维、特性与技能，作为最终形态分析的风险补充。

缺少特性或技能资料时，系统仍可基于六维给出基础判断，并明确降低解释置信度。

## 资料选择规则

- 精灵选择器默认隐藏 X 阶成长形态和首领形态。
- 最终形态和可培养命名变体可以直接选择。
- 所选形态决定页面展示的基础六维。
- 与所选最终形态匹配的首领形态参与综合分析，但首领形态不是独立培养对象。
- 技能正文只从「技能资料」读取，精灵行只保存 `skillRefs`，不复制技能长文本。

## 判断层级

### 1. 六维分布

系统计算总和、均值、相对强弱、最高项、最低项、攻击差和耐久结构。相对分布用于识别主攻、速度线、均衡双攻、单防突出和耐久短板。

### 2. 技能与特性证据

系统汇总物攻、魔攻和状态技能数量，结合攻击技能平均威力、路线占比与效果标签形成轻量证据。效果标签覆盖先手、速度、回复、减伤、能量、强化、削弱、控制、应对、轮转、连击、蓄力和场地等机制。

技能数量和公式辅助指标只用于定位校准，不视为实战伤害模拟。互斥技能、血脉选择和具体配招在资料缺少稳定分组字段时只作为风险提示。

### 3. 定位识别

定位由六维、特性和技能共同决定，主要包括：

- 物攻输出、魔攻输出、双攻输出。
- 高速先手、条件速度、后手路线。
- 耐久站场、物理防御手、魔法防御手。
- 回复辅助、异常控制、能量循环、换入返场和强化成长。

单一标签不能独立决定定位；相互冲突的证据会降低推荐强度并进入风险说明。

### 4. 候选评价

强化项服务核心定位、提高主攻、关键速度或有效耐久时获得正向理由。弱化明确弃用的攻击项通常属于低代价牺牲。

以下情况会降低分档或形成风险：

- 弱化主攻、关键速度、生命或站场所需防御。
- 低速非速度定位为了加速而牺牲核心项。
- 站场定位继续降低当前耐久短板。
- 双攻证据不足时牺牲仍有实际用途的另一攻击项。
- 只由少量通用技能触发的机制被误当成核心定位。

同一强化项内存在明显更低代价的牺牲项时，较差候选受支配并进入不推荐。弃用攻击项形成的低代价牺牲优先于普通专项防御或速度豁免。

功能型双攻、低输出功能位和护盾/免死触发路线可以保留有明确玩法依据的分支，但不会仅凭面板接近或标签数量进入推荐。

## 分档

- 推荐：强化核心定位，牺牲项代价低，没有明显硬风险，也没有被同强化项更优候选支配。
- 可保留：具有明确玩法依据，但存在场景依赖、资料不完整或可接受风险。
- 不推荐：损害核心能力、放大关键短板、缺乏定位依据，或被同强化项明显更优候选支配。

`可保留` 面向实际捕捉取舍，允许保守保留有合理玩法的分支，但不收纳没有证据的劣解。

## 速度与耐久

- 速度收益取决于基础速度、速度定位、先手/迅捷机制和牺牲代价。
- 低速功能位的加速只在低代价且有明确节奏收益时保留。
- 高速输出位可以同时存在主攻强化与速度强化两种有效方向。
- 生命同时影响两侧承伤；单防强化需要结合生命、另一防御和站场机制判断。
- 专项防御可以形成保留分支，但不能掩盖弱化明显弃用攻击项的更优选择。

## PVE 培养投入

PVE 评级先判断精灵本体机制，再从推荐或可保留性格中选择培养方向。评级证据包括：

- 高面板且路线集中的输出能力。
- DOT、百分比伤害、真实伤害或稳定层数机制。
- 能量循环、队伍插件、强化传递、站场续航和控制价值。
- 攻击技能占比、平均威力和机制在 Boss 场景中的适用性。

推荐性格数量不直接决定培养价值。高速主攻位可以同时展示速度和主攻方向；普通功能或防御站场默认按需判断，只有明确高回报机制才进入更高投入档。

DOT 证据必须命中明确的持续伤害机制并达到足够覆盖度；泛化层数文本、少量通用中毒或火系技能不能单独构成 DOT 定位。

## 输出与解释

- 候选按推荐、可保留、不推荐分组。
- 候选详情展示修正前后六维、个性理由、风险、定位和技能摘要。
- PVE 卡片展示五档星级、机制标签、主属性、主性格和可用备选方向。
- 完整技能、速度线和规则原因默认可折叠查看。

## 维护与验证

规则调整必须：

1. 更新本文件中的当前规则描述。
2. 运行 `npm run check:nature`，检查 `artifacts/nature/calibration-report.md`。
3. 对 `confirmed-results.md` 中的确认结果做回归检查。
4. 按影响范围运行 `npm test`、`npm run lint` 和 `npm run build`。
5. 将尚未形成稳定规则的问题记录到 `open-issues.md`；解决后从问题文件移除。

不为单只精灵添加硬编码特判。外部资料只用于定位和机制核对，不直接提供最终推荐性格，也不使用旧网页游戏资料作为洛克王国世界的定位证据。
