# 性格规则未决问题

本文件只记录尚未形成稳定规则、且需要跨精灵验证或用户确认的问题。已解决问题应从这里移除；稳定结论写入 `rules.md`，用户确认的单只结果写入 `confirmed-results.md`，实现历史由 Git 保存。

## 登记条件

- 同类偏差至少出现在多只精灵上，或用户明确认为它属于通用玩法口径。
- 问题无法仅通过修正文案、资料字段或单只数据解决。
- 问题可能影响推荐 / 可保留 / 不推荐分档、PVE 投入评级或广泛阈值。

## 当前问题

### 技能耗能与跨技能组合价值尚未进入路线评分

- 精灵范围：全部依赖技能循环、条件增伤或状态技铺垫的精灵；蹦床松鼠是当前观察样例。
- 现象：基础耗能数值只随技能资料传入和展示，不参与威力/耗能效率、回能循环或配招成本计算。攻击技能的强化、连击等额外效果只按标签提高路线分，条件倍率与具体幅度不折算；状态技能与攻击技能之间没有建立“铺垫 → 受益技能”的组合关系。
- 当前规则：按物理/魔法/状态数量、攻击技能基础平均威力、路线占比和效果标签做轻量定位。能量获得、能量削减和能耗变化只形成能量循环标签；状态强化、削弱、场地和蓄力只提供机制线索。
- 需要确认的口径：是否增加不依赖完整战斗模拟的轻量模型，包括基础威力/耗能效率、可识别条件倍率的保守折算、状态技对物攻/魔攻/全技能的定向支持，以及组合成立所需的能量和回合成本。
- 预期影响：可能改变物攻/魔攻路线质量差、单攻证据、速度/生命的能量循环收益和 PVE 培养评级；属于通用规则调整，不能按单只精灵特判。
- 验证样例：蹦床松鼠的魔法高威力技能与物理低耗回能技能、防反/水泡盾/化劲等状态强化组合；后续再补充至少一只低耗连发和一只高耗蓄力精灵作交叉验证。

新增问题时使用以下字段：精灵范围、现象、当前规则、需要确认的口径、预期影响和验证样例。
