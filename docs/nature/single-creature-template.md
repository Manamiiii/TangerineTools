# 单只精灵性格核对模板

用于后续“一只精灵一轮”的人工核对，避免每次输出结构漂移。复制本模板后按本轮精灵填写；不要把未确认结论写入 `docs/nature/confirmed-results.md`。

## 输出格式要求

- 默认采用“分歧优先”格式：如果用户已经在工具里看到了推荐 / 可保留 / 不推荐分组，回复中不要重复列出全部 30 个候选，也不要照抄工具已有的分档。
- 只补充工具不容易直接表达的信息：本地资料与外部定位核对、模型能力分析、PVE 投入判断、工具结果中可能存在的差异、需要用户确认的玩法口径、是否需要改规则。
- 只有当用户明确要求“全量核对 / 展开所有候选 / 为什么每个性格这么分”时，才按旧版完整格式列出全部 30 个候选。
- 如果本轮只是初步分析，也要把“状态”标为草稿，并保留“需要用户确认的玩法口径”和“规则影响记录”，避免把未确认结论误写入确认台账。
- 外部资料只做定位核对；最终捕捉建议必须以本地六维、技能线索、公式辅助指标和本工具规则为主，不直接照抄外部性格推荐。
- “推荐性格”不自动等于“值得投入 PVE 资源”；若用户讨论异色/炫彩培养，必须额外给出 PVE 培养投入判断，并明确 PVP 属性自动平衡、不作为培养资源决策依据。

## 基本信息

- 精灵：`NO.xxx 名称（形态）`
- 本轮触发原因：`工具结果有分歧 / 用户指定复核 / 全量核对 / 记录通过`
- 核对日期：`YYYY-MM-DD`
- 状态：`草稿 / 用户已确认 / 需要规则调整后复核`

> 洛克王国世界 PVP 会自动平衡精灵属性，因此“培养投入”默认只讨论 PVE。性格推荐先判断捕捉是否值得保留；若精灵是异色/炫彩，再额外判断是否值得投入 PVE 资源。

## 规则影响记录

- 是否发现规则偏差：`否 / 是`
- 偏差类型：`低速加速 / 双攻转单攻 / 降短板防御 / 公式辅助冲突 / 支配关系 / PVE 展示 / 其他`
- 是否需要立即改规则：`否；累计观察 / 是；用户已确认通用问题`
- 若暂不改，登记到：`docs/nature/rule-iteration-log.md`
