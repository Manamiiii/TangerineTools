# 性格规则迭代台账

本文件用于记录单只精灵核对中发现的通用规则问题，避免“先累计、再统一修”的结论在后续 session 中遗忘。

## 迭代原则

- 单只精灵暴露的问题先登记为观察项，不因一只精灵做硬编码特判。
- 当同类问题在多只精灵上重复出现，或用户明确确认这是通用玩法口径时，再修改规则。
- 修改规则后必须运行 `npm run check:nature`，并按影响范围考虑是否运行 `npm run audit:rocom`。
- 已确认精灵结论见 `docs/nature-confirmed-results.md`；规则调整后需要回归这些结论。

## 问题分类

| 类型 | 说明 | 典型处理 |
|---|---|---|
| 双攻转单攻 | 双攻面板接近，但技能质量、能耗或成长机制偏一侧。 | 让技能质量 / 公式辅助参与单攻分支判断，但保留双攻风险提示。 |
| 低速加速 | 基础速度低于竞争圈，但技能含控制/先手线索导致加速被高估。 | 对低速非高速定位增加加速软上限。 |
| 降短板防御 | 站场精灵降低当前短板防御仍被保留过宽。 | 同强化项存在低代价牺牲项时更容易判支配。 |
| 支配关系 | 同强化项里更低代价性格明显存在，但低优先项仍保留过多。 | 收紧支配豁免，保留“捕捉保守”但避免明显劣解。 |
| 公式辅助冲突 | 技能标签数量偏一侧，但公式辅助输出线提示另一侧技能质量更高。 | 公式辅助只做轻量投票，不替代技能/外部定位。 |
