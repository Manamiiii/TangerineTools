# 洛克王国精灵定位外部核对计划

> 目的：分批核对外部攻略/百科/社区对精灵定位的描述，与本工具的资料倾向标签、综合定位和性格推荐规则做对照，沉淀可复查的校准依据。

## 核对原则

1. **以本仓库官方同步数据为计算基准**：外部资料只用于理解玩家定位与玩法口径，不直接替代 `public/presets/rockKingdomRows.json` / `public/presets/rockKingdomSkillRows.json`。
2. **分批推进**：每批 10-20 只，优先选择校准样例、争议样例和高使用/高讨论精灵。
3. **记录来源与日期**：每个结论需要记录搜索日期、来源链接、关键词和摘要，避免把临时环境评价写成永久规则。
4. **先找规则问题，再讨论权重**：优先定位标签来源、阈值、解释文案和分档门槛是否合理；不做单只精灵手工特判。
5. **遇到口径冲突就暂停确认**：如果外部资料分歧很大，或资料明显过时，需要先和维护者确认再调整规则。

## 每只精灵记录模板

| 字段 | 说明 |
|---|---|
| 精灵 | 名称 / 编号 |
| 本地六维 | 生命、物攻、魔攻、物防、魔防、速度 |
| 本地标签 | `traitTags`、技能效果标签 Top 项 |
| 本地综合定位 | `inferRoles()` 的前几项、权重、来源理由 |
| 当前推荐摘要 | 推荐 / 可保留 / 不推荐的关键性格 |
| 外部定位摘要 | 外部攻略/百科/社区对输出、速度、耐久、辅助、传递等定位的描述 |
| 差异判断 | 标签过强、标签缺失、阈值问题、技能识别问题、解释问题、无明显差异 |
| 建议动作 | 不改 / 报告解释 / 标签来源拆分 / 阈值调整 / 分档门槛调整 / 需要人工讨论 |
| 来源 | URL、查询关键词、访问日期 |

## 批次安排

### 第 1 批：当前校准争议样例

- 音速犬：高速物攻输出，验证少量防御/后手技能不应推导减速或单防推荐。
- 冰钻布鲁斯：验证“物防高但生命低”的肉度判断，重点核对是否应被视为脆皮输出而非耐久站场。
- 天鹅类强化传递样例：验证慢速交接、强化传递与生命/速度性格的解释口径。
- 寂灭骨龙等后手/应对样例：验证真正后手收益与普通 PVP 工具箱的差异。

### 第 2 批：高物防 / 高魔防但低生命样例

目标是校验“单防高是否足以生成耐久标签”。本批重点看：

- 生命低于中位数但单防达到 P75/P90 的精灵；
- 外部评价是否仍认为其脆；
- `+物防` / `+魔防` 是否应默认只到可保留。

### 第 3 批：高生命 / 综合肉度样例

目标是验证 `生命 + 双防` 综合阈值是否能正确识别真正耐久站场。

### 第 4 批：速度关键节点样例

目标是核对速度档位和关键锚点：哪些精灵必须保速，哪些精灵可以低速/后手。

## 当前规则待重点核对的问题

1. `defense` 标签不应只由单项物防/魔防高触发；肉度定位更应依赖生命 + 双防综合基础。
2. 生命很低但单防高的精灵，应避免直接把单防性格推成“推荐”。
3. 报告中的“特性标签”需要继续弱化误解：部分标签来自六维阈值，不一定来自特性文本机制。
4. 速度已经有锚点解释；生命、物防、魔防也需要通过全局分布与关键阈值继续校准，但不应简单复制速度锚点逻辑。
