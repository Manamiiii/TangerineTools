# 外部研究资料源

## 定位

TangerineTools 使用两类资料源：

- **正式结构化源**：BWiki staging / preview / apply 管线负责编号、名称、形态、六维、系别、特性、技能关系等运行时预置字段。
- **外部研究源**：B站、小红书等玩家内容用于理解版本玩法、实战定位、养成路线、队伍作用、机制演示和玩家分歧。

外部研究材料是分析证据，不是正式预置字段。帖子或视频中的主观结论不得直接覆盖 BWiki 正式数据，也不得直接变成性格推荐结果。

## 平台用途

| 平台 | 适合核对的内容 | 使用方式 |
|---|---|---|
| [B站](https://www.bilibili.com/) | 战斗录像、技能演示、首领进化前后差异、养成与配队讲解、明确版本时间的视频攻略 | 记录视频链接、标题、UP 主、发布时间、相关分段时间和可复核画面 |
| [小红书](https://www.xiaohongshu.com/) | 单只精灵图文攻略、一图流、养成面板、队伍组合、版本体验和评论区补充 | 记录笔记链接、标题、作者、发布时间或更新时间、相关图片页和结论摘要 |

平台内容可能需要登录后查看。研究时可以使用用户已经登录的浏览器会话进行人工检索和阅读；不绕过登录、验证码、访问限制或反自动化措施，不批量抓取账号内容、评论和用户信息。

## 证据记录

每次单只精灵研究至少保存以下信息：

1. 精灵名称、形态和研究问题。
2. 来源平台、原始链接、标题、作者和发布时间。
3. B站视频的相关时间点，或小红书笔记的相关图片页 / 正文位置。
4. 可观察事实与作者观点的区分。
5. 内容对应的游戏版本；无法确认版本时明确标记。
6. 与本地六维、特性、技能关系或既有规则的相符点和冲突点。

页面标题、链接和简短摘要可以写入研究结果；视频、图片和长篇正文不复制进仓库。用户确认后的稳定结论写入 `docs/nature/confirmed-results.md`，尚未解决的规则问题写入 `docs/nature/open-issues.md`。

## 采用顺序

同一结论存在多个来源时，按以下顺序判断：

1. 游戏内可复核画面或明确数值。
2. 与当前版本一致、展示完整操作过程的材料。
3. 多个独立作者能够相互印证的结论。
4. 单一作者的分析或评论区观点。

发现外部材料与 BWiki 正式字段冲突时，先作为数据源问题记录并回到 BWiki staging 核对；未经正式 preview / apply 流程，不修改运行时预置。发现多个精灵共同暴露同一推荐规则问题时，按性格推荐工作流请求用户确认后再修改通用规则。

## 检索范围

检索词默认包含“洛克王国世界”、精灵完整名称和具体问题，例如“PVE”“特性”“技能”“首领形态”“性格”“配队”。不使用旧页游《洛克王国》内容作为《洛克王国：世界》的机制或定位依据，除非任务明确要求历史对比。
