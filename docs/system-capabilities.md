# TangerineTools · 系统能力说明

本文档描述当前代码库已经实现的能力范围，以及明确排除在范围外的部分。目的是让任何人（包括未来的开发 session）一眼看清"现在能做什么、还不能做什么"。

## 定位

TangerineTools 是一个**本地优先（local-first）**的个人资料管理 Web App：

- 所有数据保存在浏览器 IndexedDB（通过 Dexie.js 访问），不依赖任何后端服务或云账号。
- 纯静态站点，可直接部署到 GitHub Pages / Cloudflare Pages 等静态托管平台。
- 面向个人使用（Mac / Android 浏览器），不含多用户、鉴权、协作等能力。
- 数据备份/迁移通过手动 JSON 导出/导入完成。

## 技术栈

- Vite + React 19
- Dexie.js 4（`dexie-react-hooks` 的 `useLiveQuery` 做响应式读取）
- lucide-react 图标
- 无 CSS 框架，纯手写 `src/styles.css`
- 无路由库，`App.jsx` 里基于 `window.location.hash` 实现极简路由
- oxlint 做 lint（`npm run lint`）

## 已实现能力

### 1. 场景管理（首页）

- 场景以「行」列表展示（非卡片），每行显示：色点、名称、类型、已启用工具、编辑/删除按钮。
- 支持新建 / 编辑 / 删除场景（弹窗表单）。
- 场景字段：名称、类型（通用整理/游戏资料/镜头素材/资料档案，`constants.js` 的 `SCENE_TYPES`）、色调（`COLOR_PALETTE` 预设色板，新建时优先随机未使用颜色）、启用的工具（`SCENE_TOOLS`）。
- 点击场景行进入该场景的工作台（hash 路由 `#scene/{id}`）。

场景工作台按 `scene.tools` 里启用的、且 `ready: true` 的工具渲染。若启用了多个工具，工作台顶部会出现分段式切换器，可在已启用工具之间自由切换（不会互相影响数据）。目前 `catalog`（资料库）、`owned`（收集记录）、`stock`（统计视图）、`nature`（性格推荐）四个工具均已落地。

### 2. 资料库工具（`catalog`）

在场景工作台内，若场景启用了「资料库」工具，会渲染 `CatalogTool`：

- **资料表（table）管理**：一个场景可以有多个资料表；支持新建 / 重命名 / 删除（仅剩一个表时禁止删除）；顶部有表选择器切换当前查看的表。
- **字段管理**：支持新增、删除、重命名、改类型、拖拽排序、显示/隐藏字段。共 11 种字段类型（`FIELD_TYPES`）：
  - 短文本 / 长文本 / 数字 / 图片 / 单选 / 多选 / 布尔 / URL / 日期 / 引用资料 / 指标视图
  - 「单选/多选」支持配置选项（文本 + 点击展开的颜色选择 + 可选图标），并可拖拽调整选项顺序。
  - 「引用资料」会在行编辑表单中从指定资料表选择一条记录，表格/详情中展示被引用行的文本标签；点击标签可打开被引用资料的详情弹窗。
  - 「指标视图」（字段类型仍为 `stats`）是派生视图字段，本身不可编辑，可用条形列表或雷达图展示；通过 `statsDimensions`/`statsMap` 关联任意数量的底层数值字段，支持自动识别（按字段名/key 别名）或手动指定映射。洛克王国预置为生命/物攻/魔攻/物防/魔防/速度六项，并默认使用条形列表。
- **列头菜单**：每列表头都有「列操作」菜单——编辑、向左插入列、向右插入列、筛选、排序（升/降）、隐藏、删除。
- **筛选面板**：按字段类型提供不同的筛选控件（数字/日期用范围，单选/多选用勾选列表，布尔用二态开关，指标视图暂不支持筛选）。
- **排序**：点击列头可切换升序/降序；默认按「编号」字段自然升序排列（若无编号字段则不做特殊排序）。
- **分页**：可调页大小（10/20/50/100，默认 20），显示总条数与页码。
- **表格视图（紧凑）**：单元格文本两行截断 + 悬浮提示；操作列固定在右侧、表头固定在顶部，方便横向/纵向滚动时操作。
- **详情视图（宽松）**：点击「查看详情」弹出详情弹窗，展示该行**全部字段**（含被隐藏的底层数值字段，并标注「隐藏列」徽章），提供编辑/删除入口；若该资料表存在「编号」字段，详情弹窗还会展示**同编号形态对比**区块（`FormComparisonSection`），把编号相同、形态不同的行并排列出各数值维度并标注每个维度里的最高/最低/相同值；此外还会为每一行给出**适合方向**（如「物攻输出/魔攻输出/双攻输出/高速先手/耐久坦克/能量循环/辅助续航/异常控制/…侧重」等一句话定位，由六维分布 + 特性标签驱动）和**主要差异**（相对同组均值的最强/最弱 1~2 个维度概括），便于一眼比较同一编号下不同形态的取向。
- **搜索**：工具栏内的搜索框对可搜索字段做文本匹配过滤。

### 3. 收集记录工具（`owned`）

场景启用「收集记录」工具后会渲染 `OwnedTool`，用来记录“我和资料库对象之间的收集关系”。它同时覆盖两种模型：

- **一对一**：同一个资料项只保留一条收集记录，适合成就、镜头、素材、角色解锁等“一项只需要完成/收集一次”的内容。
- **一对多**：同一个资料项可以有多条收集记录，适合洛克王国精灵、随机装备、重复卡牌等“一种对象可以拥有多份实例”的内容。

实现约束：

- 数据复用 `catalogTables` / `catalogFields` / `catalogRows` 存储，通过 `catalogTables.kind === 'owned'` 与普通资料库表区分；`collectionMode` 是非索引配置，不引入 Dexie schema 版本变更。
- 普通新建场景首次进入「收集记录」时只创建一张空的收集记录表，不预置任何业务字段；用户可在该工具内打开「字段」自行配置。
- 洛克王国是应用自带的预置场景，首次进入「收集记录」时会补齐精灵收集字段（精灵引用、性格、血脉、是否炫彩、特长、性别、备注），这些字段被视为洛克王国场景自身的数据模板，而不是所有场景的默认模板。
- 单元格渲染、表单输入复用资料库的 `CellView` / `FieldInput`，视觉风格与资料库一致。
- 一对一模式下，如果存在 reference 字段，新增同一引用对象的记录会更新已有记录，避免重复；一对多模式允许同一引用对象出现多条记录。

### 4. 统计视图工具（`stock`）

场景启用「统计视图」工具后会渲染 `StockTool`。它不再为新场景创建固定字段表，而是从当前场景已有的资料库表与收集记录表中选择数据源，按字段做分组统计，并可叠加一个数值阈值条件。

- 数据源：当前场景内的普通资料表（`!table.kind`）与收集记录表（`kind === 'owned'`）。
- 分组：选择任意非指标视图字段；多选字段会展开到多个分组。
- 条件：可选择一个数值字段并设置“≥ 阈值”的过滤条件。
- 结果：展示总记录数、命中记录数，以及各分组数量。
- 旧版本遗留的 `kind === 'stock'` 表不会被自动删除，导入导出仍会保留这些数据；但新 UI 不再依赖该固定表。

### 5. 性格推荐工具（`nature`）

场景启用「性格推荐」工具后会渲染 `NatureTool`：

- 六维数值支持手动输入，也可以点击「带入」从场景内任意一张**普通资料表**（即 `!table.kind`，不含统计视图/收集记录）中选择一行，自动带出名称、六维数值（按 `STATS_DIMENSIONS` 别名识别）、特性标签。
- 根据六维数值与特性标签计算推荐的性格（强化一项数值、弱化另一项数值），并展示加成前后的六维对比表格（强化/弱化维度高亮标注）。
- **候选方案列表**：除了展示评分最高的“推荐”性格，工具还会同时列出评分次高的若干“可选”候选（默认 top-6），点击可切换查看每个候选性格的加成对比与推荐理由，便于在近似分之间做取舍。
- 展示可解释的推荐理由文案（`explainNatureRecommendation`），而非仅给出结论。
- 六维全部为空（或都为 0）时展示空态提示，不强行计算。

### 6. 洛克王国预置资料

- 首次启动自动播种（`ensureSeeded`，通过 `meta` 表的 `seededRockKingdom` 标记防止重复播种）：
  - 场景「洛克王国」（游戏类型，新安装默认启用「资料库」「收集记录」「统计视图」「性格推荐」四个工具，进入后直接显示四工具切换器，默认停留在资料库；旧版本曾经启用不同组合，`ensureSeeded` 里的 `migrateRockKingdomSceneTools` 会在场景 `tools` 仍恰好等于任一旧默认组合（`['catalog']` 或 `['catalog', 'stock', 'nature']`）时自动补齐为四项，一旦用户自定义过 `tools` 则不覆盖），资料表「精灵基础资料」，18 个字段（含 6 个隐藏的六维底层字段）。
  - 496 条官方图鉴精灵行，数据由 `scripts/sync-rock-kingdom-preset.mjs` 从 `https://static.gamecenter.qq.com/xgame/roco-kingdom/compendium/d.json` 采集：遍历 `l` 基础条目并展开详情里的 `forms` 独立形态，生成 `public/presets/rockKingdomRows.json` 后运行时 `fetch` 加载；若离线加载失败，场景/表/字段骨架依然可用，只是没有行数据。
  - 系别字段使用 `multiselect` 类型，覆盖官方全部 18 个系别（普通/草/火/水/光/地/冰/龙/电/毒/虫/武/翼/萌/幽/恶/机械/幻），图标使用洛克王国官方图鉴静态资源（`https://static.gamecenter.qq.com/xgame/roco-kingdom/compendium/a/e/<系别中文名>.png`），支持多系精灵；`form` 字段是自由文本（例如「基础形态」「进化形态」「究极形态」），`shiny` 是 `yes/no` 单选。
  - 精灵图、系别图标、特性图标均使用同源公开静态资源前缀 `https://static.gamecenter.qq.com/xgame/roco-kingdom/compendium/`，中文路径按段编码；不使用本地 SVG 占位精灵图。
  - 迁移策略：老用户升级到新版本时，`migrateRockKingdomRows()` 会删除默认资料表中明确可识别的旧 `row-rock-*` / `data:image/svg+xml` 占位行，再插入官方稳定 id 行；用户新增的非占位资料行、owned 收集记录、stock 统计视图不会被删除；`meta.rockKingdomRowsVersion` 记录资料版本，不引入 Dexie schema 版本变更。

### 7. 全量数据导出 / 导入（仅首页）

- **导出**：一键导出全部 Dexie 数据（`scenes` / `catalogTables` / `catalogFields` / `catalogRows` / `meta`）为带时间戳的 JSON 文件下载。
- **导入**：选择 JSON 文件后二次确认，采用「同 id 覆盖、文件中不存在的本地数据保留」的合并策略（非整表替换）；导入前做基础结构校验（`validateImportPayload`）。


### 8. Android Chrome PWA 安装基础能力

- 站点提供 Web App Manifest、`theme-color`、192/512 SVG 图标和 `display: standalone` 配置。
- Manifest 的 `start_url`、`scope` 和图标路径使用相对路径，配合 Vite `base: './'` 兼容 GitHub Pages 仓库子路径部署。
- 提供一个极简 service worker 用于满足 Android Chrome 安装条件；它会在安装/激活后尽快接管页面，但不预缓存资源、不返回自定义缓存响应、不读写 IndexedDB，避免影响用户本地数据或让旧版本因运行时缓存长期滞留。

## 明确排除在范围外的能力

以下能力经过评估后**明确不在当前范围内**，不是遗漏或缺陷：

- **洛克王国对局向深度资料**：当前预置资料只覆盖公开图鉴 `d.json` 可映射的基础资料、六维、特性、图片、系别、形态；技能、进化链、属性克制、PVP 规则等仍不在范围内。
- 技能/进化链/属性克制等游戏对局向功能、PVP 相关工具、AI 自动打标签、云同步/多用户协作、后端服务：均超出"个人本地资料管理工具"的定位，未纳入规划。

新 session 若要继续扩展上述范围外的能力，请先阅读 `docs/session-start-prompt.md`。

## 本地开发命令

```bash
npm install       # 安装依赖
npm run dev       # 启动开发服务器
npm run build     # 生产构建（构建产物在 dist/）
npm run lint      # oxlint 静态检查
npm run preview   # 预览 build 产物
```

已验证：`npm run build`、`npm run lint` 均通过；并通过浏览器手动走查确认了场景列表、场景导航、多工具切换（资料库/收集记录/统计视图/性格推荐 四工具切换器）、资料表格渲染（搜索/排序/筛选/分页，496 条洛克王国预置行的滚动/翻页表现）、收集记录增删改查与统计（血脉/炫彩）、统计视图分组统计、性格推荐的资料库带入与候选切换、详情弹窗（含同编号形态对比 + 适合方向 + 主要差异列）等核心链路可正常工作，详见 `docs/session-start-prompt.md` 的"已验证的质量基线"。
