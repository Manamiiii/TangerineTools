# TangerineTools · 系统能力说明

本文档描述当前代码库已经实现的能力范围，以及明确排除在范围外的部分。目的是让任何人（包括未来的开发 session）一眼看清"现在能做什么、还不能做什么"。

## 定位

TangerineTools 是一个**本地优先（local-first）**的个人资料管理 Web App：

- 所有数据保存在浏览器 IndexedDB（通过 Dexie.js 访问），不依赖任何后端服务或云账号。
- 纯静态站点，可直接部署到 GitHub Pages / Cloudflare Pages 等静态托管平台。
- 面向个人使用（Mac / Android 浏览器），不含多用户、鉴权、协作等能力。
- 数据备份/迁移通过手动 JSON 导出/导入完成。

## 技术栈

- Vite + React 19
- Dexie.js 4（`dexie-react-hooks` 的 `useLiveQuery` 做响应式读取）
- lucide-react 图标
- 无 CSS 框架，纯手写 `src/styles.css`
- 无路由库，`App.jsx` 里基于 `window.location.hash` 实现极简路由
- oxlint 做 lint（`npm run lint`）

## 已实现能力

### 1. 场景管理（首页）

- 场景以「行」列表展示（非卡片），每行显示：色点、名称、类型、已启用工具、编辑/删除按钮。
- 支持新建 / 编辑 / 删除场景（弹窗表单）。
- 场景字段：名称、类型（通用/游戏/资料，`constants.js` 的 `SCENE_TYPES`）、色调（`COLOR_PALETTE` 八色板）、启用的工具（`SCENE_TOOLS`）。
- 点击场景行进入该场景的工作台（hash 路由 `#scene/{id}`）。

场景工作台按 `scene.tools` 里启用的、且 `ready: true` 的工具渲染。若启用了多个工具，工作台顶部会出现分段式切换器，可在已启用工具之间自由切换（不会互相影响数据）。目前 `catalog`（资料库）、`stock`（属性库存）、`nature`（性格推荐）三个工具均已落地。

### 2. 资料库工具（`catalog`）

在场景工作台内，若场景启用了「资料库」工具，会渲染 `CatalogTool`：

- **资料表（table）管理**：一个场景可以有多个资料表；支持新建 / 重命名 / 删除（仅剩一个表时禁止删除）；顶部有表选择器切换当前查看的表。
- **字段管理**：支持新增、删除、重命名、改类型、拖拽排序、显示/隐藏字段。共 11 种字段类型（`FIELD_TYPES`）：
  - 文本 / 长文本 / 数字 / 图片 / 单选 / 多选 / 布尔 / URL / 日期 / 引用 / 六维图
  - 「单选/多选」支持配置选项（文本 + 颜色 + 可选图标）。
  - 「六维图」是派生视图字段，本身不可编辑，通过 `statsMap` 关联到六个底层数值字段（生命/物攻/魔攻/物防/魔防/速度），支持自动识别（按字段名/key 别名）或手动指定映射。
- **列头菜单**：每列表头都有「列操作」菜单——编辑、向左插入列、向右插入列、筛选、排序（升/降）、隐藏、删除。
- **筛选面板**：按字段类型提供不同的筛选控件（数字/日期用范围，单选/多选用勾选列表，布尔用二态开关，六维图暂不支持筛选）。
- **排序**：点击列头可切换升序/降序；默认按「编号」字段自然升序排列（若无编号字段则不做特殊排序）。
- **分页**：可调页大小（10/20/50/100，默认 20），显示总条数与页码。
- **表格视图（紧凑）**：单元格文本两行截断 + 悬浮提示；操作列固定在右侧、表头固定在顶部，方便横向/纵向滚动时操作。
- **详情视图（宽松）**：点击「查看详情」弹出详情弹窗，展示该行**全部字段**（含被隐藏的底层数值字段，并标注「隐藏列」徽章），提供编辑/删除入口；若该资料表存在「编号」字段，详情弹窗还会展示**同编号形态对比**区块（`FormComparisonSection`），把编号相同、形态不同的行并排列出各数值维度，并标注每个维度里的最高/最低/相同值。
- **搜索**：工具栏内的搜索框对可搜索字段做文本匹配过滤。

### 3. 属性库存工具（`stock`）

场景启用「属性库存」工具后会渲染 `StockTool`，用于登记一批"实例"（例如正在培养的精灵）并做简单统计：

- 数据复用资料库的 `catalogTables` / `catalogFields` / `catalogRows` 存储，通过 `catalogTables.kind === 'stock'` 标记与普通资料表区分（首次进入该工具时由 `ensureStockTable` 幂等创建），因此资料库工具的表选择器、性格推荐工具的"从资料库带入"选择器都会自动过滤掉这张库存表，互不干扰；沿用现有的全量导出/导入逻辑，无需额外处理。
- 字段是**固定**的 5 个（非用户可配置）：名称（文本）、等级（数字）、分类（文本）、状态（单选：未开始/培养中/已完成）、备注（长文本）。
- 支持新增 / 编辑 / 删除实例；单元格渲染、表单输入复用资料库的 `CellView` / `FieldInput`，视觉风格与资料库一致。
- **统计视图**（工具栏「统计」按钮切换显示/隐藏）：
  - 按分类分组计数（空分类归入"未分类"，按数量降序）。
  - 按状态计数（未开始/培养中/已完成三个固定状态各自的数量）。
  - 数值型条件统计：等级 ≥ 可调阈值的实例数量。
- 未做字段管理、排序、筛选、分页（固定字段 + 单表场景下暂不需要）；这些是可预期的简化，不是缺陷。

### 4. 性格推荐工具（`nature`）

场景启用「性格推荐」工具后会渲染 `NatureTool`：

- 六维数值支持手动输入，也可以点击「带入」从场景内任意一张**非库存**资料表中选择一行，自动带出名称、六维数值（按 `STATS_DIMENSIONS` 别名识别）、特性标签。
- 根据六维数值与特性标签计算推荐的性格（强化一项数值、弱化另一项数值），并展示加成前后的六维对比表格（强化/弱化维度高亮标注）。
- 展示可解释的推荐理由文案（`explainNatureRecommendation`），而非仅给出结论。
- 六维全部为空（或都为 0）时展示空态提示，不强行计算。

### 5. 洛克王国预置资料（演示数据）

- 首次启动自动播种（`ensureSeeded`，通过 `meta` 表的 `seededRockKingdom` 标记防止重复播种）：
  - 场景「洛克王国」（游戏类型，预置只启用了「资料库」工具；可在编辑场景弹窗里勾选启用「属性库存」「性格推荐」以体验多工具切换），资料表「精灵基础资料」，18 个字段（含 6 个隐藏的六维底层字段）。
  - 10 条示例精灵行（迪莫、喵喵、魔力猫、水灵、圣水守护、鸭吉吉、鸭吉吉国王、烈焰虎、烈焰霸王、磐石龟，含 4 组同编号的进化形态对），数据来自 `public/presets/rockKingdomRows.json`，运行时 `fetch` 加载（避免把图片占位数据打进主 bundle）；若离线加载失败，场景/表/字段骨架依然可用，只是没有行数据。
  - 系别/形态/特性标签的选项图标使用纯本地内联 SVG data URI 占位，不引用任何外部美术资源。
  - 这 10 条是**示例/演示数据**，不是完整的洛克王国精灵图鉴（完整图鉴规模远大于此）；扩充为全量数据是一项独立的、明确排除在当前范围外的工作，详见 `docs/data-sync.md` 的说明与未来接入方式。

### 6. 全量数据导出 / 导入（仅首页）

- **导出**：一键导出全部 Dexie 数据（`scenes` / `catalogTables` / `catalogFields` / `catalogRows` / `meta`）为带时间戳的 JSON 文件下载。
- **导入**：选择 JSON 文件后二次确认，采用「同 id 覆盖、文件中不存在的本地数据保留」的合并策略（非整表替换）；导入前做基础结构校验（`validateImportPayload`）。

## 明确排除在范围外的能力

以下能力经过评估后**明确不在当前范围内**，不是遗漏或缺陷：

- **洛克王国全量数据同步**：当前预置数据是 10 条示例/演示数据，不是完整图鉴；补齐全量数据依赖真实的官方数值资料，且体量较大，作为独立任务另行规划（数据结构与未来接入方式见 `docs/data-sync.md`）。
- **PWA 安装配置**（离线缓存、manifest、可安装到主屏幕等）：这是一个**主动排除**的范围决策，而非未完成的缺陷——当前定位是"可直接用浏览器打开的本地优先工具"，是否需要可安装体验留待后续按需评估。
- 技能/进化链/属性克制等游戏对局向功能、PVP 相关工具、AI 自动打标签、云同步/多用户协作、后端服务：均超出"个人本地资料管理工具"的定位，未纳入规划。

新 session 若要继续扩展上述范围外的能力，请先阅读 `docs/session-start-prompt.md`。

## 本地开发命令

```bash
npm install       # 安装依赖
npm run dev       # 启动开发服务器
npm run build     # 生产构建（构建产物在 dist/）
npm run lint      # oxlint 静态检查
npm run preview   # 预览 build 产物
```

已验证：`npm run build`、`npm run lint` 均通过；并通过浏览器手动走查确认了场景列表、场景导航、多工具切换（资料库/属性库存/性格推荐）、资料表格渲染（搜索/排序/筛选/分页）、属性库存增删改与统计、性格推荐带入与推荐结果、详情弹窗（含同编号形态对比）等核心链路可正常工作，详见 `docs/session-start-prompt.md` 的"已验证的质量基线"。
