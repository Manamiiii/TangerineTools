# TangerineTools · 系统能力说明

本文档描述当前代码库已经实现的能力范围，以及明确排除在范围外的部分。目的是让任何人（包括未来的开发 session）一眼看清"现在能做什么、还不能做什么"。

## 定位

TangerineTools 是一个**本地优先（local-first）**的个人资料管理 Web App：

- 所有数据保存在浏览器 IndexedDB（通过 Dexie.js 访问），不依赖任何后端服务或云账号。
- 纯静态站点，可直接部署到 GitHub Pages / Cloudflare Pages 等静态托管平台。
- 面向个人使用（Mac / Android 浏览器），不含多用户、鉴权、协作等能力。
- 数据备份/迁移通过手动 JSON 导出/导入完成。

## 技术栈

- Vite + React 19
- Dexie.js 4（`dexie-react-hooks` 的 `useLiveQuery` 做响应式读取）
- lucide-react 图标
- 无 CSS 框架，纯手写 `src/styles.css`
- 无路由库，`App.jsx` 里基于 `window.location.hash` 实现极简路由
- oxlint 做 lint（`npm run lint`）

## 已实现能力

### 1. 场景管理（首页）

- 场景以「行」列表展示（非卡片），每行显示：色点、名称、类型、已启用工具、编辑/删除按钮。
- 支持新建 / 编辑 / 删除场景（弹窗表单）。
- 场景字段：名称、类型（通用整理/游戏资料/镜头素材/资料档案，`constants.js` 的 `SCENE_TYPES`）、色调（`COLOR_PALETTE` 预设色板，新建时优先随机未使用颜色）、启用的工具（`SCENE_TOOLS`）。
- 点击场景行进入该场景的工作台（hash 路由 `#scene/{id}`）。

场景工作台按 `scene.tools` 里启用的、且 `ready: true` 的工具渲染。若启用了多个工具，工作台顶部会出现分段式切换器，可在已启用工具之间自由切换（不会互相影响数据）。目前 `catalog`（资料库）、`owned`（收集记录）、`stock`（统计视图）、`nature`（性格推荐）、`breeding`（孵蛋推荐）五个工具均已落地。

### 2. 资料库工具（`catalog`）

在场景工作台内，若场景启用了「资料库」工具，会渲染 `CatalogTool`：

- **资料表（table）管理**：一个场景可以有多个资料表；支持新建 / 重命名 / 删除（仅剩一个表时禁止删除）；顶部有表选择器切换当前查看的表。
- **字段管理**：支持新增、删除、重命名、改类型、拖拽排序、显示/隐藏字段。共 13 种字段类型（`FIELD_TYPES`）：
  - 短文本 / 图文摘要 / 长文本 / 数字 / 图片 / 单选 / 多选 / 布尔 / URL / 日期 / 引用资料 / 多选引用资料 / 指标视图
  - 「图文摘要」的主文字存放在当前字段，可在字段管理中关联同表图片字段和描述字段；表格、详情和编辑表单均按配置组合展示，不依赖业务字段名。
  - 「单选/多选」支持配置选项（文本 + 点击展开的颜色选择 + 可选图标），并可拖拽调整选项顺序。
  - 「引用资料」会在行编辑表单中从指定资料表选择一条记录；「多选引用资料」会选择多条记录。表格/详情中展示被引用行的文本标签；点击标签可打开被引用资料的详情弹窗。
  - 「指标视图」（字段类型仍为 `stats`）是派生视图字段，本身不可编辑，可用条形列表或雷达图展示；通过 `statsDimensions`/`statsMap` 关联任意数量的底层数值字段，支持自动识别（按字段名/key 别名）或手动指定映射。洛克王国预置为生命/物攻/魔攻/物防/魔防/速度六项，并默认使用条形列表。
- **展示配置**：字段的非索引 `display` 元数据可声明紧凑列宽、括号分行、纵向选项、可配置行数的标签、`+x` 溢出提示、摘要单元格、引用头像/标签和图标化单选。资料库与收集记录共用这套渲染协议，场景预置不需要在通用组件中增加字段名判断。新增/编辑行时，选项不超过 5 个的单选使用平铺按钮，较多选项使用下拉框。
- **列头菜单**：每列表头都有「列操作」菜单——编辑、向左插入列、向右插入列、筛选、排序（升/降）、隐藏、删除。
- **筛选面板**：按字段类型提供不同的筛选控件（数字/日期用范围，单选/多选用勾选列表，布尔用二态开关，指标视图暂不支持筛选）。
- **排序**：点击列头可切换升序/降序；洛克王国精灵表默认按「编号 -> X 阶从小到大 -> 最终形态 -> 首领形态」排列；同编号、同形态内，无括号的名称优先于带括号的名称。其他有编号的资料表仍按编号自然升序。
- **分页**：可调页大小（10/20/50/100，默认 20），显示总条数与页码。
- **表格视图（紧凑）**：单元格文本两行截断 + 悬浮提示；操作列固定在右侧、表头固定在顶部，方便横向/纵向滚动时操作。
- **详情视图（宽松）**：点击「查看详情」弹出详情弹窗，展示该行**全部字段**（含被隐藏的底层数值字段，并标注「隐藏列」徽章），提供编辑/删除入口。同编号形态的培养向对比集中在性格推荐工具中展示。
- **搜索**：工具栏内的搜索框对可搜索字段做文本匹配过滤。

### 3. 收集记录工具（`owned`）

场景启用「收集记录」工具后会渲染 `OwnedTool`，用来记录“我和资料库对象之间的收集关系”。它同时覆盖两种模型：

- **一对一**：同一个资料项只保留一条收集记录，适合成就、镜头、素材、角色解锁等“一项只需要完成/收集一次”的内容。
- **一对多**：同一个资料项可以有多条收集记录，适合洛克王国精灵、随机装备、重复卡牌等“一种对象可以拥有多份实例”的内容。

实现约束：

- 数据复用 `catalogTables` / `catalogFields` / `catalogRows` 存储，通过 `catalogTables.kind === 'owned'` 与普通资料库表区分；`collectionMode` 是非索引配置，不引入 Dexie schema 版本变更。
- 普通新建场景首次进入「收集记录」时只创建一张空的收集记录表，不预置任何业务字段；用户可在该工具内打开「字段」自行配置。
- 洛克王国是应用自带的预置场景，首次进入「收集记录」时会补齐精灵收集字段（精灵引用、性格、血脉、个体异色、是否炫彩、特长、性别、备注），这些字段被视为洛克王国场景自身的数据模板，而不是所有场景的默认模板。精灵引用按字段配置展示头像和名称；异色与血脉使用 BWiki / patchwiki 图标，炫彩使用游戏界面的多色五角星图标，性别使用经典红蓝符号；非异色与非炫彩值在表格中留空。
- 单元格渲染、表单输入复用资料库的 `CellView` / `FieldInput`，视觉风格与资料库一致；可搜索引用字段使用同一个组合框完成展开、输入筛选和选择。
- 一对一模式下，如果存在 reference 字段，新增同一引用对象的记录会更新已有记录，避免重复；一对多模式允许同一引用对象出现多条记录。

### 4. 统计视图工具（`stock`）

场景启用「统计视图」工具后会渲染 `StockTool`。它从当前场景的资料库表与收集记录表中选择数据源，按字段做即时分组统计，并可叠加一个数值阈值条件。

- 数据源：当前场景内的普通资料表（`!table.kind`）与收集记录表（`kind === 'owned'`）。
- 分组：选择任意非指标视图字段；首次打开优先选择单选、多选、布尔等分类字段，多选字段会展开到多个分组。
- 条件：可选择一个数值字段并设置“≥ 阈值”的过滤条件。
- 结果：展示总记录数、命中记录数，以及各分组数量。
- 聚合计算位于 `src/domain/stock.js`，多选字段展开、空值分组和数值阈值均由纯逻辑测试覆盖。
- 导入兼容 `kind === 'stock'` 的历史记录并原样保留；当前统计视图不读取这类固定表。

### 5. 性格推荐工具（`nature`）

场景启用「性格推荐」工具后会渲染 `NatureTool`：

- 顶部使用一个可搜索组合框完成精灵展开、输入筛选和选择；手动六维输入默认折叠，支持生命、物攻、魔攻、物防、魔防、速度。
- 洛克王国场景中，资料带入固定绑定「精灵基础资料」表。选择精灵后以紧凑摘要展示名称、形态、六维、该精灵实际具有的特性标签，并通过 `skillRefs` 读取「技能资料」表中的真实技能行；未选择精灵时不展示具体特性标签。
- 精灵选择器每个编号只展示一个最终普通形态入口；X 阶成长形态、地区/外观变体和首领形态不重复占用筛选项，但仍各自使用自己的六维、特性与技能独立判断。综合推荐取各形态结果的并集，详情标注候选性格适用的具体形态。六维条形图从 0 起算，以全资料库全部属性的真实最高值为统一满刻度，并标记全局最低有效值；顶部另外给出分位、排名和统计基线。同编号形态对比使用系别图标，在展开全部技能前标出仅某形态拥有的技能，并强调相对入口形态变化的六维数值和分位。
- 性格推荐按“同进化链精灵引用 + 具体性格值”匹配当前场景的收集记录；非最终形态的拥有记录也计入最终形态。未获得候选提供预填精灵与性格的快速新增入口。
- 单只精灵定位核对可以引用 B站和小红书的当前版本玩法材料；链接、作者、时间和具体画面位置按外部研究资料规范记录，观察事实与作者观点分开处理。这些材料只参与规则审计，不直接写入 BWiki 正式预置或照搬外部性格结论。
- 评价全部 30 个合法性格候选（6 个可修正维度中，强化一项、弱化另一项；不允许同项增减）。
- 全部候选先按强化维度分组，再按「推荐 / 可保留 / 不推荐」分档；候选行展示性格增益/代价、精确获得状态和评分。多形态精灵的候选行展示各形态评分范围，详情逐形态展示独立评级与分数，不计算跨形态平均分。
- 技能线索、公式辅助和速度线作为固定分析依据放在 PVE 与性格候选之间；形态间技能或速度不同时可展开逐形态依据。所选性格区域只展示修正、分行后的重点理由、风险与修正后六维，并对核心属性和风险词做重点强调。
- 「PVE 培养投入」按同编号全部有效形态分别给出星级、定位、主属性和性格方向，避免重复展示同义结论或没有新增信息的详细依据。
- 技能分析统计物攻/魔攻/状态技能数量、攻击技能平均威力、物攻/魔攻占比，并读取技能资料中的 `effectTags` 派生效果标签（先手、速度、回复、减伤、能量、强化、控制、应对、轮转等），用于解释速度、耐久、能量与控制相关的性格收益。未决规则问题记录在 `docs/nature/open-issues.md`。

### 6. 孵蛋推荐工具（`breeding`）

场景启用「孵蛋推荐」工具后会渲染 `BreedingTool`：

- 读取当前「收集记录」里的精灵引用、性别、是否异色、是否炫彩和性格。
- 读取「精灵基础资料」里的 `eggGroups`（蛋组）与 `speciesGroup`（繁育谱系）字段；当前正式资料来自版本化 BWiki 发布流程。`npm run sync:breeding` 生成 staging，发布流程将繁育字段合入正式精灵行。
- 运行时直接读取精灵预置中的繁育字段；三方合并会补齐或更新明确的正式字段，同时保留用户非空自定义值。
- 默认只分析拥有记录里的异色个体；只推荐一公一母且至少共享一个可孵蛋组的组合；每组最多 5 对，即 10 只精灵，默认输出最多 5 组。
- 推荐优先级按“未拥有该同种异色 > 可孵出推荐性格 > 双异色父母 > 异色炫彩父 + 普通母 > 已有父母”的口径排序。
- 后代种类按母亲提示，性格继承按父/母/随机概率说明展示；异色/炫彩父母数量只作为期望排序线索，不写死未确认的精确概率。
- 收集记录只保存用户实际新增的数据；应用不写入孵蛋演示个体。

### 7. 洛克王国预置资料

- 首次启动自动播种（`ensureSeeded`，通过 `meta.seededRockKingdom` 防止重复播种）：
  - 场景「洛克王国」默认启用资料库、收集记录、统计视图、性格推荐、孵蛋推荐五个工具。
  - 普通资料表包括「精灵基础资料」和「技能资料」。
  - 精灵基础资料包含官方基础资料、六维、图文摘要特性、精灵果实（界面标为「种子」）、精灵蛋、`traitTags`、`skillTags`、`skillRefs`，并维护 `eggGroups`、`speciesGroup`。
  - 技能资料包含技能图标、名称、系别、类型、威力、能耗、先制/速度、效果标签、效果、`learnerRefs`。
- 正式数据由版本化 BWiki staging / preview 经显式覆盖命令发布：
  - `public/presets/rockKingdomRows.json` 包含 592 条精灵 / 形态资料。
  - `public/presets/rockKingdomSkillRows.json` 包含 553 条技能资料。
  - `public/presets/rockKingdomPresetMigration.json` 保存基线正式值 SHA-256，供浏览器做安全三方合并，不含用户数据。
- 精灵与技能图片使用经审计的 BWiki / patchwiki URL；UI 图标使用可信静态资源。
- 迁移策略：通过版本化基线正式值指纹做三方合并，只更新空值、无效值或匹配基线正式值的精灵 / 技能字段；用户自定义非空值、用户新增的非占位资料行、owned 收集记录和 stock 统计配置不会被覆盖或删除；不改变 Dexie schema 版本。

### 8. 全量数据导出 / 导入（仅首页）

- **导出**：一键导出全部 Dexie 数据（`scenes` / `catalogTables` / `catalogFields` / `catalogRows` / `meta`）为带时间戳的 JSON 文件下载。
- **导入**：选择 JSON 文件后二次确认，采用「同 id 覆盖、文件中不存在的本地数据保留」的合并策略（非整表替换）；导入前做基础结构校验（`validateImportPayload`）。


### 9. Android Chrome PWA 安装基础能力

- 站点提供 Web App Manifest、`theme-color`、192/512 SVG 图标和 `display: standalone` 配置。
- Manifest 的 `start_url`、`scope` 和图标路径使用相对路径，配合 Vite `base: './'` 兼容 GitHub Pages 仓库子路径部署。
- 提供一个极简 service worker 用于满足 Android Chrome 安装条件；它在安装/激活后接管页面，不预缓存资源、不返回自定义缓存响应、不读写 IndexedDB，避免影响用户本地数据或造成运行时缓存滞留。

## 明确排除在范围外的能力

以下能力经过评估后**明确不在当前范围内**，不是遗漏或缺陷：

- **洛克王国对局向深度功能**：BWiki 技能资料用于资料查看和性格推荐输入，不提供完整对战模拟、技能组合求解、属性克制、PVP 环境或配队工具。
- AI 自动打标签、云同步/多用户协作、后端服务：均超出“个人本地资料管理工具”的定位，未纳入规划。

## 本地开发命令

```bash
npm install       # 安装依赖
npm run dev       # 启动开发服务器
npm run build     # 生产构建（构建产物在 dist/）
npm run lint      # oxlint 静态检查
npm run preview   # 预览 build 产物
```
