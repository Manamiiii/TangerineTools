1. 先看每个样例的“技能摘要”和“效果标签计数”是否符合直觉。
2. 再按“强化生命/物攻/魔攻/物防/魔防/速度”分组横向比较全部 30 个性格，优先找同一强化项里哪个弱化项被误判。
3. 最后看“推荐 / 可保留 / 不推荐”的最终分档是否符合该精灵定位。
4. 如果某类样例普遍偏差，再回到 `src/domain/nature.js` 调整对应权重。
5. 调整后重新运行 `npm run check:nature`、`npm run build`、`npm run lint`。
